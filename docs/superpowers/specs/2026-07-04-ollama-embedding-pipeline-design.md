# Ollama Embedding Pipeline — Design Spec

**Date:** 2026-07-04
**Stage:** Embedding pipeline robustness (pre-search)

---

## Context

The project has a working but rough embedding script (`src/db/embedding.ts`) that calls Ollama's `nomic-embed-text` model to generate 768-dimension vectors for ~6,800 books and stores them in a pgvector column. The script works, but has hardcoded config, no startup health check, silent per-book failures, no vector index, and is not registered as a npm script. This spec hardens the pipeline before the next stage (semantic search endpoint) is built.

---

## Goals

- Make the embedding script safe to run on 6K+ books on CPU without silent failures
- Expose all tunable config via `.env`
- Validate Ollama is reachable before doing any work
- Collect and summarise failed book IDs at the end (no retry — recoverable by re-run)
- Add a cosine HNSW index so the column is query-ready for the next stage
- Lay a clean module boundary so switching LLM providers later requires a new module, not a rewrite of the runner

---

## Architecture

```
src/db/
  ollama.ts       ← NEW: Ollama transport (waitForOllama, getEmbedding)
  embedding.ts    ← UPDATED: runner only (loop, failed IDs, summary)
  schema.ts       ← UPDATED: HNSW index declaration
drizzle/
  <new>.sql       ← generated by bun db:generate, applied by bun db:migrate
.env / .env.example ← UPDATED: OLLAMA_URL, OLLAMA_MODEL, EMBEDDING_CHUNK_SIZE
package.json      ← UPDATED: db:embed script
src/db/ollama.test.ts ← NEW: unit tests
```

**Provider-swap pattern:** `embedding.ts` imports `{ waitForOllama, getEmbedding }` from `./ollama`. A future `openai.ts` exports the same interface — swapping means changing one import line in the runner.

---

## Module Designs

### `src/db/ollama.ts` (new)

Exports:

```ts
waitForOllama(opts?: { maxAttempts?: number; delayMs?: number }): Promise<void>
getEmbedding(text: string): Promise<number[]>
```

**`waitForOllama`**
- Reads `OLLAMA_URL` from env (default `http://localhost:11434`)
- Polls `GET /` on the base URL
- Default: 10 attempts, 2000 ms apart (20 s total budget)
- Logs `Waiting for Ollama… (attempt N/10)` on each failure
- Throws `Error("Ollama not reachable after N attempts")` if all attempts fail
- Caller (`main()`) catches this and aborts with a clear message

**`getEmbedding`**
- Reads `OLLAMA_URL` and `OLLAMA_MODEL` from env (defaults: `http://localhost:11434`, `nomic-embed-text`)
- POSTs `{ model, prompt }` to `<OLLAMA_URL>/api/embeddings`
- Throws a descriptive error on non-2xx response
- Returns `number[]`

---

### `src/db/embedding.ts` (updated)

Responsibilities:
1. Call `waitForOllama()` — abort if it throws
2. Read `EMBEDDING_CHUNK_SIZE` from env (default `20`)
3. Loop: fetch batch of books with `isNull(embedding)`, call `getEmbedding(createBookSearchText(book))`, update row
4. On per-book error: push `book.id` to `failedIds[]`, log inline, continue
5. After loop: print summary — total processed, total failed, list of failed IDs

`createBookSearchText()` stays in this file (book-domain logic, not LLM transport).

---

### `src/db/schema.ts` (updated)

Add HNSW cosine index on the `embedding` column using Drizzle's `index()` API. Drizzle will generate the migration SQL:

```sql
CREATE INDEX ON books USING hnsw (embedding vector_cosine_ops);
```

HNSW chosen over IVFFlat: no row-count pre-computation needed, faster query time at this scale.

## Environment Variables

| Variable | Default | Description |
|---|---|---|
| `OLLAMA_URL` | `http://localhost:11434` | Base URL for Ollama service |
| `OLLAMA_MODEL` | `nomic-embed-text` | Embedding model name |
| `EMBEDDING_CHUNK_SIZE` | `20` | Books processed per batch (raise to 50 with GPU) |

Added to both `.env` and `.env.example`.

---

## Scripts

`package.json`:
```json
"db:embed": "bun src/db/embedding.ts"
```

Migration workflow (run once after schema change):
```bash
bun db:generate   # writes new SQL migration to drizzle/
bun db:migrate    # applies migration to running Postgres container
```

---

## Unit Tests — `src/db/ollama.test.ts`

Uses Bun's built-in test runner (`bun test`). Network calls mocked via `spyOn(global, 'fetch')`.

| Test | Assertion |
|---|---|
| `waitForOllama` — Ollama ready on first poll | resolves without error |
| `waitForOllama` — succeeds on attempt N (< max) | resolves, logs N-1 retries |
| `waitForOllama` — exhausts all attempts | throws with descriptive message |
| `getEmbedding` — success | returns correct `number[]` |
| `getEmbedding` — non-200 response | throws with status in message |
